# Project Charter: AthleticaOS (Rugby Module)

## 1. Vision
To create a unified, national-grade rugby management platform for Malaysia that digitises the entire ecosystem—from grassroots to national levels—serving as the foundation for a future multi-sport operating system (AthleticaOS).

## 2. Mission
To digitise and centralise data for players, teams, clubs, associations, and tournaments under the AthleticaOS umbrella, ensuring data integrity, accessibility, and operational efficiency for the Malaysia Rugby Union and its affiliates.

## 3. Scope

### Phase 1 (Current)
- **Architectural Design**: Defining the system structure, database schema, and technology stack.
- **Documentation**: Producing comprehensive SRS, FRD, SysRD, and Data Dictionary.
- **ERD Refinement**: Finalising the data model for the rugby module.
- **Planning**: Establishing timelines and resource requirements.

### Overall System (Future Phases)
- **Backend**: Spring Boot application with PostgreSQL.
- **Frontend**: React + Vite + TypeScript web application.
- **Mobile**: Responsive design / PWA for on-field usage.
- **Portals**: Admin, Team Manager, Tournament Organiser, Public.
- **Analytics**: Player performance and match statistics.

## 4. Stakeholders
- **Malaysia Rugby Union (MRU)**: Strategic oversight and national data ownership.
- **State Unions**: Regional management and governance.
- **Clubs & Schools**: Team management and player development.
- **Coaches & Officials**: Match management and performance tracking.
- **Tournament Organisers**: Event scheduling and results management.
- **Players**: Profile management and career history.

## 5. Constraints
- **Resource**: Single developer availability (weekends only).
- **Hardware**: Development on MacBook Pro M2 with limited RAM (8GB).
- **Budget**: Minimal initial infrastructure cost (local dev first).
- **Timeline**: Extended timeline due to part-time development.

## 6. Success Criteria (Phase 1)
- Completion of all specified documentation (Charter, SRS, FRD, SysRD, Data Dictionary, ERD, UX Flows, Timeline, Architecture).
- User approval of the architectural foundation.
- Readiness for Phase 2 (Coding) commencement.
